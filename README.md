## 一、为什么有这份 SOP

### 历史问题
1. 批量出图时 render prompt 组装脚本里被自加 `max_chars=3800` 硬截断 · 导致信息删减
2. 公司视觉规范白皮书（4884 字符）被复制到每页头部 · 导致冗余暴涨
3. 高密度页（>8000 字符）走 image model · 模型客观漏字
4. transform 行为不上报 · 失去可审计性

### SOP 解决目标
- **禁止 silent transform**：任何替换 / 截断 / 摘要必须显式记录
- **物理隔离 4 层产物**：源文件 / 样式层 / render prompt / 审计报告
- **统一拼装公式**：精简样式 + 完整源文 · 一字不删
- **按密度自动分流**：低密度图像 / 高密度仍用图像但拉高分辨率
- **强制审计报告**：每页 diff_report.json 落盘

## 四、执行原则（不可绕过）

1. **不得删减任何正文行**
2. **不得压缩列表为摘要**
3. **不得省略任何数字 / 术语 / 标签 / 卡片标题**
4. **不得做术语替换**（如需替换 · 必须输出替换清单到 diff_report.json）
5. **不得截断**（如超模型上限 · 必须 raise FAIL · 不得自行精简）
6. **任何 transform 必须在 diff_report.json 中显式记录**
