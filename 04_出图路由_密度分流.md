# 04 · 出图路由 · 密度分流（v1.1）

## 一、最终路由规则（v1.1 · 仅 2 档）

| 真实正文字符 | 分辨率 | 模型 | 风险等级 |
|---|---|---|---|
| ≤ 5,000 字符 | **2k** | gpt-image-2 | low / medium |
| > 5,000 字符 | **4k** | gpt-image-2 | high / extreme |

**关键变更（v1.0 → v1.1）：**
- ❌ 移除 1k 档（最低分辨率统一提升到 2k）
- ❌ 移除 v1.0 中"5000-8000 用 2k / >8000 用 4k"的三档划分
- ✅ 收敛为 2 档：**5K 以下 2k · 5K 以上 4k**

## 二、为什么取消 1k 档

1. **中文密字页 1k 必糊**：即使 ≤3000 字，1k 分辨率下中文笔画会糊成方块
2. **2k 是中文清晰度下限**：微软雅黑 12pt 以下字号 · 1k 几乎不可读
3. **统一最低标准**：避免因字符波动导致同一份 PPT 出现清晰度断层

## 三、为什么 5K 以上强制 4k

1. **信息密度阈值**：5K 字符 = 一页 PPT 的高密度上限
2. **>5K 必出现小字号块**：注释/来源/卡片副标题 ≤14px · 需要 4k 才能锐利
3. **避免漏字**：4k 给 image model 更大的字符空间 · 减少压缩漏失

## 四、字符数计算口径

- **统计对象**：源文件 markdown 纯文本字符数（含中文 · 含标点 · 不含 frontmatter）
- **不含**：380 字符样式声明 · 包装层 · 视觉规范引用
- **示例**：
  - L1_P7 源文 2,260 字符 → **2k**
  - L1_P19 源文 7,943 字符 → **4k**
  - L1_P26 源文 25,301 字符 → **4k**（已达模型上限 · 同时触发结构锁定）

## 五、强结构页特殊规则（v1.1 新增）

即使字符数 ≤5,000，若页面属于以下类型之一，**强制升档到 4k**：

- 多区块硬约束页（A/B/C/D/E 五区结构等）
- 附图 100% 复刻页（用户指定保留原版）
- 多时序流页（波形 / 时间轴 / 三流并列）
- 流程/拓扑/架构图页（节点+连线密集）

**判定标准**：源文出现 `区域 A` / `区域 B` / `保持原版` / `100% 继承` 等结构锁定关键词。

## 六、调用模板

```python
def route(real_chars: int, is_structure_locked: bool = False) -> str:
    if is_structure_locked:
        return "4k"
    if real_chars <= 5000:
        return "2k"
    return "4k"
```

## 七、与上游/下游的衔接

- **上游**：03_强制规则 · 不得自行降档（如 5K 以上强行用 2k）
- **下游**：05_diff_report_schema · 必须记录 image_size 与 density_risk
- **审计**：06_工程师执行checklist · 出图后核验分辨率匹配

## 八、反模式（写入 99）

- ❌ 用 1k 出中文密字页（必糊）
- ❌ ≤3000 字就用 1k（v1.0 旧规则 · 已废止）
- ❌ 5K-8K 用 2k（v1.0 旧规则 · 已废止）
- ❌ 强结构页只看字符数路由（必须升档 4k）
