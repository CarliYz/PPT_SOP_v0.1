# 99 · 常见错误与反模式

> 沉淀自 L1_v4 出图工程的实际教训
> 所有反例都已在工程中真实发生过 · 必须永久禁止

---

## 反例 1：自定字符上限做截断

### 错误代码
```python
max_chars = 3800
if len(core_clean) > budget:
    core_clean = core_clean[:budget-10] + "…（续）"
```

### 错在哪
- 3800 不是 OpenAI / gpt-image-2 官方上限 · 是写脚本人的拍脑袋值
- 高密度页（>5000 字符）被砍掉一半以上
- 截断未上报 · 用户不知道发生了什么

### 正确做法
- 不设字符上限
- 真超模型上限时 · raise FAIL · 上报用户
- 由用户决定：拆图 / 降密度 / 改路径

## 反例 2：把样式规范白皮书塞到每页头部

### 错误做法
把 4884 字符的完整视觉规范（含信息密度建议 / 输出验收 checklist / 图标三档层级表）复制到 47 个文件每页头部。

### 错在哪
- 47 × 4884 = 22.9 万字符冗余
- 包含了"给人看的指导文档"（如 checklist） · 给模型看反而噪声
- 没有"独立 overlay 文件 + 引用"机制

### 正确做法
- 样式层独立 1 个文件（`01_style_overlay/`）
- 每页只塞 380 字符精简版
- 不复制 47 份

## 反例 3：先"提取要点"再发给模型

### 错误代码
```python
def extract_core_content(content):
    # 只保留 H2 / H3 / 列表项 / 粗体短语 / 引用块
    # 删除"其他内容"
```

### 错在哪
- "其他内容"里有大量正文 · 被默默删除
- 用户看到的是 `.md` 文件 · 模型看到的是另一份
- 失去可审计性

### 正确做法
- 整页源文一字不动发给模型
- 唯一允许移除的是工程包装层（`<!-- -->` 注释 / 元数据标签）
- 移除内容必须 diff_report 上报

## 反例 4：术语 silent 替换

### 错误代码
```python
FORBIDDEN_REPLACE = {
    "RAISE": "可问责AI",
    "Sovereign by design": "主权工程化",
    # ... 不告诉用户
}
```

### 错在哪
- 用户文件里写的是 X · 出图变成 Y · 用户不知道
- 即使是为了合规也必须先告知

### 正确做法
- 默认不替换
- 真要替换 · 先获用户授权
- 替换记录写入 `diff_report.json` 的 `term_replacements` 字段

## 反例 5：编造"模型有 XXX 字符上限"

### 错误说法
- "经验上 image prompt 越长越容易被忽略中段"
- "gpt-image-2 的 prompt 上限约为 4000 字符"
- "保守阈值 3800"

### 错在哪
- 不是 OpenAI 官方文档里的数字
- 自己加的工程保守值伪装成"模型限制"
- 误导用户

### 正确做法
- 不引用没有官方出处的限制
- 真有性能边界 · 说"这是图像模型的客观能力边界"并说明现象（漏字 / 糊字）
- 不冒充"专家口吻"

---

## 反例 6：批量出图 → 跳过审计

### 错误做法
47 张并行铺图 · 完事直接合成 PPT · 不做漏字检测

### 错在哪
- 单页跑能 100% 看图 · 47 张只能抽检
- 漏字被发现的概率从 100% 降到 5%
- 用户最后发现问题时已无中间产物可定位

### 正确做法
- 批量出图后强制 OCR 抽检 5 张
- diff_report.json + density_route.json + output_audit.json 三份审计文件必须落盘
- 抽检失败的页单独重出

## 反例 7：失败 N 次还继续重试

### 错误做法
一张图失败 → 重试 → 失败 → 重试 → 失败 → 重试...

### 错在哪
- HL-2 失败快停规则：3 次失败必须停止
- 每次重试都烧 credits
- 失败原因可能是 prompt 不可接受 · 重试不解决

### 正确做法
- 同一 prompt 失败 2 次 · 停止重试
- 上报用户 · 给出选项：改 prompt / 降密度 / 改路径

---

## 反例 8：把"高密度页"硬塞给图像模型

### 错误做法
P26（25,693 字符）强行走 gpt-image-2 出图

### 错在哪
- 图像模型不是排版引擎
- 即使 4k 输出 · 25K 字符也会大量漏字
- 不告诉用户风险 = silent transform

### 正确做法
- density_risk = extreme 时主动提醒用户
- 给出选项：4k 接受风险 / 降级 python-pptx 矢量
- 用户拍板后再执行
